System flow and integration overview.

About this diagram

This flowchart outlines the end‑to‑end journey of an application. The UI collects data, triggers orchestration, and receives a real‑time decision with per‑check details for transparency.

High‑level steps

  1. Submit application: The user completes the form and clicks “Apply Now”.
  2. Orchestration: A coordinator dispatches parallel checks and aggregates updates.
  3. Parallel checks: AML, Fraud, Credit, and Income run concurrently and stream results.
  4. Eligibility decision: Results map to a tier (0–7) and credit limit rules.
  5. Report + telemetry: The UI shows a summary, request ID, and technical details.

Key components

  • SEON (Fraud): Device/behavioral signals, risk indicators (mock).
  • Experian (Credit): Bureau tiering and scorecard hints (mock).
  • Plain (Income): Payroll/bank mode, coverage months, stability (mock).
  • Taktile (Decision): Policy composition into Pass/Review/KO and tier limits (mock).

Note: This is a demo representation for product exploration; data, endpoints, and policies are mocked.
